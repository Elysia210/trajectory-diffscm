Trajectory Diff-SCM — Apr11 Rerun & Physical Constraints

Experiment report · June 2026 · Ruimin Zhang

Summary. The trajectory Diff-SCM baseline was rerun end-to-end on the updated Apr11 dataset, and physical constraints were added both at sampling time and in training. The recommended setup — the acceleration-regularized diffusion model with sampling-side guidance guidance_scale=5, lambda_step=2, lambda_accel=10 — produces a strong counterfactual collision push (mean collision-probability increase ≈ 0.29) while keeping motion close to physically feasible (≈ 12% of steps above the 8 m/s² cap, versus ≈ 50% for the original unguided model). A key diagnostic finding: the unguided diffusion model itself produced infeasible accelerations, so the durable fix had to be in training, not only in sampling.

Task

The request was to (1) rerun Diff-SCM on the updated trajectory data and (2) add physical constraints during sampling if possible. Both are addressed; (2) was extended to a training-side fix after a diagnostic finding.

1. Data integration (Apr11)

All 14,638 ego/adversary pair-scenes from the 189-file export load correctly with zero skipped (schema and 100-step length match; per-sample feature tensor [100, 23]). No config change beyond the data path was needed.

The export carries no collision labels, so binary labels were generated geometrically (rotated bounding-box overlap via the Separating-Axis Theorem): 2,203 collision / 12,435 non-collision. A labeling pitfall was caught and fixed: scene keys repeat across files (14,638 rows vs only 4,841 unique keys), so the label manifest was rekeyed by the globally unique source_file:scene_key, after which all 14,638 scenes matched their own label.

2. Pipeline fixes

Classifier input normalization. The classifier initially did not learn (validation PR-AUC stuck at chance) because it consumed raw features whose world-coordinate positions (hundreds–thousands of metres) saturated the GRU. Per-feature standardization (stored as model buffers, so the sampler self-normalizes too) fixed it: validation PR-AUC 0.99, F1 0.97. The step-diffusion model trained cleanly (validation loss → 0.052).

3. Physical constraints added

  • Feature recomputation (sampling). After a position intervention, heading, velocity, and relative-motion features are recomputed from the generated positions so the classifier scores a self-consistent trajectory (previously they were stale).
  • Acceleration hinge (sampling). A feasibility cap penalizing only acceleration above a threshold (default 8 m/s²), rather than forcing the trajectory to copy the reference.
  • Acceleration regularizer (training). A smoothness/acceleration penalty on the model’s predicted step sequence, applied only in the low-noise regime (see §5).

4. Sampling-side results

5. Key finding — the model itself is the bottleneck

The unguided reference itself is infeasible (≈ 224 m/s² peak, ≈ 50% of steps over the cap). Sampling constraints clean up the guided output, but the realism ceiling is set by the diffusion model, which was trained without any smoothness regularization. This motivated a training-side fix.

6. Training-side regularization (root-cause fix)

A first naive attempt (regularizing all timesteps, weight 1.0) backfired: at high diffusion noise the predicted clean sample is essentially noise with huge meaningless acceleration, so the term dominated training, wrecked fidelity (val 0.21 vs 0.052), and made the model’s own samples worse. Restricting the regularizer to the low-noise regime (timesteps ≤ 20% of the schedule) fixed this:

7. Combined (recommended) configuration

Regularized model + sampling hinge (lambda_accel=10): guided over-threshold fraction 0.12 (vs 0.16 on the un-regularized model) at a collision-probability increase of 0.29. The two mechanisms are complementary — training lowers the model’s intrinsic infeasibility, sampling enforces a per-sample physical cap.

8. Conclusions

  • The new Apr11 data is fully integrated and the end-to-end Diff-SCM pipeline runs on it.
  • Physical constraints work at two levels: a sampling-side acceleration cap, and a training-side regularizer that makes the model’s own prior more feasible — without hurting denoising fidelity or counterfactual push.
  • The recommended setup gives a strong, controllable counterfactual collision push with roughly four-fold fewer physically-infeasible steps than the original baseline.
  • Methodological takeaway worth keeping: a regularizer on the predicted clean sample must be gated to the low-noise regime, or it competes with — and loses to — the denoising objective.

9. Next steps

  • Push the training regularizer slightly harder (fidelity has slack to ≈ 0.08) to lower the model’s intrinsic infeasibility further.
  • The trajectory version currently starts from random noise rather than inverting a real future (no Abduction step), so it is “Diff-SCM-inspired” rather than a strict counterfactual. Implementing trajectory DDIM inversion is the natural way to make it a true counterfactual — a good topic for the next meeting.
